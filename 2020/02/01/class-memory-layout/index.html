最近在家闲的快发霉了，无聊之下花了点功夫把部分代码在OSX上跑起来(之前都是在Win上开发)，遇到了一个很有意思的崩溃：最后定位出来是不同compiler下的memory layout区别。

首先从表象上来看，加载同一份资源在Win上没问题，然后OSX上稳定崩溃(都是x64)；先干掉各种异步加载接口和构造最下可复现例子之后，对比发现是某个类的数据很奇怪：

直观的看就是mNbVertices和mNbTriangles错位了，同时后面的mVertices开始又没问题。接下来找下这块数据是哪里赋值的：

这里使用了in placement new，先排查下address部分的数据，对比了下OSX/Win下没区别。然后看下具体出问题的父类定义：

以及打印下这些基类的大小和成员变量地址

其中sizeof的结果两个平台一致

256 160 16 16 16

但是地址部分出现了不一致

  • OSX上的成员变量地址偏移
    • mNbVertices 28
    • mNbTriangles 32
    • mVertices 40
    • mRefCount 24
  • Win上的成员变量地址偏移
    • mNbVertices 32
    • mNbTriangles 36
    • mVertices 40
    • mRefCount 24

OK这里的结果也就是最前面的调试结果一致了。临时解决方案是RefCountable里加了个int占位。

但这里有点奇怪的是明明PxTriangleMesh和RefCountable都是大小为16，那么TriangleMesh的两个基类应该直接占掉32了，为什么XCode里mNbVertices居然是28…
